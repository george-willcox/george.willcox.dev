Introduction:

There are many ways available to calculate \(\pi\), but a common way is by use of an infinite sum. This involves adding a set of numbers together that all follow a set pattern, and then sometimes multiplying or dividing the result by another number to get \(\pi\). Here, there are two different methods of calculating \(\pi\) shown.

The first of the two methods shown, is the simple but inefficient infinite sum that was used by James Gregory and Gottfried Leibniz, which is a series expansion of the arctangent function. By using the fact that \(\text{arctan}(1) = \frac{\pi}{4}\) they found the series:

\[ \pi = 4\left ( \frac{1}{1} - \frac{1}{3} + \frac{1}{5} - \frac{1}{7} + \frac{1}{9} - \cdots \right ) \]

Then the second of these sequences is a similar infinite sum that was proposed by Sir Isaac Newton. He expanded the arcsine function, and used the fact that \(\text{arcsin}(1/2) = \frac{\pi}{6}\) to obtain the series:

\[ \pi = 6 \left (\frac{1}{2^1 \cdot 1} + \left ( \frac{1}{2} \right ) \frac{1}{2^3 \cdot 3} + \left ( \frac{1 \cdot 3}{2 \cdot 4} \right ) \frac{1}{2^5 \cdot 5} + \left ( \frac{1 \cdot 3 \cdot 5}{2 \cdot 4 \cdot 6} \right ) \frac{1}{2^7 \cdot 7} + \cdots \right ) \]

This arcsine series converges much faster than using the arctangent, only 22 terms are needed to obtain the first 16 decimal places of \(\pi\). Whereas by using the arctangent series, 30000 iterations are not enough to calculate the 5th decimal place.

Arctangent Series:

The source code for the arctangent series can be downloaded here: Arctangent series.zip

Now, although this series is very inefficient, it is very simple, and so the code is very simple too. This makes it a perfect introduction for working with infinite sums and sequences.

To start with we will create two variables, one to store the value of \(\pi\) that we calculate, and one to store the iteration that we are currently on. 

n = 0
value = 0

The next thing to do after this, is to create a while loop that calculates each term in the sequence. As the numerator of every term is always the same, that is quite easy to do. Also, the denominator increases as a linear sequence, which can be written as 2n + 1 (starting with n = 0) What also needs to be kept in mind is how the terms alternate adding and subtracting, so on each iteration of the loop we will add the next two terms. 

while True:
    value += 1 / (2 * n + 1)
    n += 1
    value -= 1 / (2 * n + 1)
    n += 1

You can see here that we have increased the variable n by 1 each time we have calculated a new term, and we simply add the terms together to calculate the value of \(\pi\). This is what the value variable is used for. When adding the terms, make sure to alternate adding and subtracting so that an accurate value for \(\pi\) is calculated.

Now that we have the code set up to calculate \(\pi\), we just need to print the result. For ease I am going to print the result after every two terms, so that is once for every iteration of the loop. When doing this, we must remember that \(\pi\) is equal to 4 times the sum of all of the terms in the sequence. This means that when printing the result to the screen, we will have to multiply the value variable by 4. 

print(4 * value)

And there we have it, a program that can calculate \(\pi\) al the way up to 15 decimal places. It would be possible to increase the number of decimal places that we calculate \(\pi\) to, but this is unnecessary due to the inefficiency of the series. Since it takes an incredible large number of iterations to calculate it to any substantial degree of accuracy, it is unlikely that the program will ever manage to reach 15 decimal places.

Here is the finished code in its entirety: 

n = 0
value = 0

while True:
    value += 1 / (2 * n + 1)
    n += 1
    value -= 1 / (2 * n + 1)
    n += 1

    print(4 * value)
